class cerebro.keras.SparkEstimator(model=None, custom_objects=None, optimizer=None, loss=None, batch_size=None, loss_weights=None, sample_weight_col=None, metrics=None, callbacks=None, shuffle_buffer_size=None, transformation_fn=None, model_update_fn=None)¶

Cerebro Spark Estimator for fitting Keras model to a DataFrame.

Supports tf.keras >= 2.2.

Parameters

  • model – Keras model to train.

  • custom_objects – Optional dictionary mapping names (strings) to custom classes or functions to be considered during serialization/deserialization.

  • optimizer – Keras optimizer.

  • loss – Keras loss or list of losses.

  • batch_size – Number of rows from the DataFrame per batch.

  • loss_weights – (Optional) List of float weight values to assign each loss.

  • sample_weight_col – (Optional) Column indicating the weight of each sample.

  • metrics – (Optional) List of Keras metrics to record.

  • callbacks – (Optional) List of Keras callbacks.

  • model_update_fn – (Optional) Function that can be used to update a Keras model (e.g., freeze/unfreeze layers) after every epoch. Takes in a Keras model and epoch number as input and returns a potentially updated Keras model.

class cerebro.tune.GridSearch(backend, store, estimator_gen_fn, search_space, num_epochs, evaluation_metric='loss', validation=0.25, label_columns=['label'], feature_columns=['features'], verbose=1)¶

Performs grid search using the given param grid

Parameters

  • backend – Cerebro backend object (e.g., SparkBackend).

  • store – Cerebro store object (e.g., LocalStore, HDFSStore).

  • estimator_gen_fn – A function which takes in a dictionary of parameters and returns a Cerebro Estimator (e.g., cerebro.SparkEstimator).

  • search_space – A dictionary object defining the parameter search space.

  • num_epochs – Number of maximum epochs each model should be trained for.

  • evaluation_metric – Evaluation metric used to pick the best model (default “loss”).

  • validation – (Optional) The ratio of the validation set (default: 0.25) or a string defining the column name defining the validation set. In the latter case the column value can be bool or int.

  • label_columns – (Optional) A list containing the names of the label/output columns (default [‘label’]).

  • feature_columns – (Optional) A list containing the names of the feature columns (default [‘features’]).

  • verbose – Debug output verbosity (0-2). Defaults to 1.

class cerebro.tune.TPESearch(backend, store, estimator_gen_fn, search_space, num_models, num_epochs, evaluation_metric='loss', validation=0.25, label_columns=['label'], feature_columns=['features'], parallelism=None, verbose=1)¶

Performs Tree of Parzen Estimators (TPE) search using the given param grid.

Parameters

  • backend – Cerebro backend object (e.g., SparkBackend).

  • store – Cerebro store object (e.g., LocalStore, HDFSStore).

  • estimator_gen_fn – A function which takes in a dictionary of parameters and returns a Cerebro Estimator (e.g., cerebro.SparkEstimator).

  • search_space – A dictionary object defining the parameter search space.

  • num_models – Maximum number of models to be searched.

  • num_epochs – Number of maximum epochs each model should be trained for.

  • evaluation_metric – Evaluation metric used to pick the best model (default loss).

  • validation – (Optional) The ratio of the validation set (default: 0.25) or a string defining the column name defining the validation set. In the latter case the column value can be bool or int.

  • label_columns – (Optional) A list containing the names of the label/output columns (default [‘label’]).

  • feature_columns – (Optional) A list containing the names of the feature columns (default [‘features’]).

  • parallelism – (Optional) Number of models trained in parallel. If not specified will default to Cerebro number of workers. If model training times vary significantly consider increasing the number (e.g., 2*num_workers) for better cluster utilization.

  • verbose – Debug output verbosity (0-2). Defaults to 1.

class cerebro.backend.SparkBackend(spark_context=None, num_workers=None, start_timeout=600, disk_cache_size_gb=10, data_readers_pool_type='thread', num_data_readers=10, nics=None, verbose=1)¶

Spark backend implementing Cerebro model hopping

Parameters

  • spark_context – Spark context

  • num_workers – Number of Cerebro workers. Defaults to spark.default.parallelism.

  • start_timeout – Timeout for Spark tasks to spawn, register and start running the code, in seconds. If it is not set as well, defaults to 600 seconds.

  • disk_cache_size_gb – Size of the disk data cache in GBs (default 10GB).

  • data_readers_pool_type – Data readers pool type (‘process’ or ‘thread’) (default ‘thread’)

  • num_data_readers – Number of data readers (default 10)

  • nics – List of NIC names, will only use these for communications. If None is specified, use any available networking interfaces (default None)

  • verbose – Debug output verbosity (0-2). Defaults to 1.

prepare_data(store, dataset, validation, num_partitions=None, parquet_row_group_size_mb=8, dataset_idx=None)¶

Prepare data by writing out into persistent storage

Parameters

  • store – Cerebro storage object (e.g., LocalStorage, HDFSStorage).

  • dataset – Spark DataFrame.

  • validation – Fraction of validation data (e.g., 0.25) or name of the DataFrame column indicating validation.

  • num_partitions – Number of data partitions of the output. If None, will default to the current number of input dataset partitions.

  • parquet_row_group_size_mb – Parquet row group size in MBs (default 8 MB) .

  • dataset_idx – Dataset index if storing multiple datasets in the same directory.

class cerebro.storage.HDFSStore(prefix_path, train_path=None, val_path=None, runs_path=None, temp_dir=None, host=None, port=None, user=None, kerb_ticket=None, driver='libhdfs', extra_conf=None)¶

Uses HDFS as a store of intermediate data and training artifacts.

Initialized from a prefix_path that can take one of the following forms:

  1. “hdfs://namenode01:8020/user/test/Cerebro”

  2. “hdfs:///user/test/Cerebro”

  3. “/user/test/Cerebro”

Parameters

  • prefix_path – Prefix path of the local directory (e.g., /user/test/cerebro).

  • train_path – (Optional) Path of the directory to store training data. If not specified will default to <prefix_path>/train_data

  • val_path – (Optional) Path of the directory to store validation data. If not specified will default to <prefix_path>/val_data

  • runs_path – (Optional) Path of the directory to store model checkpoints and log. If not specified will default to <prefix_path>/runs

  • temp_dir – (Optional) Temp directory on the local machine.

  • host – (Optional) NameNode hostname.

  • port – (Optional) NameNode port.

  • user – (Optional) Username connecting to HDFS.

  • kerb_ticket – (Optional) Path to Kerberos ticket cache.

  • driver – (Optional) Driver to be used for HDFS communication (default ‘libhdfs’)

  • extra_conf – (Optional) Extra Key/Value pairs for config; Will override any hdfs-site.xml properties

class cerebro.storage.LocalStore(prefix_path, train_path=None, val_path=None, runs_path=None)¶

  • val_path – (Optional) Path of the directory to store validation data. If not specified will default to <prefix_path>/val_data

  • runs_path – (Optional) Path of the directory to store model checkpoints and log. If not specified will default to <prefix_path>/runs
